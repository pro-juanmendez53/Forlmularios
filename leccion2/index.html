Lección 2: Estructura Básica de un Formulario

La etiqueta <form>

Todo formulario en HTML comienza y termina con la etiqueta <form>. Esta etiqueta actúa como un contenedor para todos los elementos de entrada (campos de texto, botones, etc.) que componen el formulario.

La etiqueta <form> tiene dos atributos principales que son cruciales para su funcionamiento: action y method.

Atributo action

El atributo action especifica la URL a la que se enviarán los datos del formulario cuando el usuario lo envíe. Esta URL generalmente apunta a un script en el servidor (escrito en lenguajes como PHP, Python, Node.js, etc.) que procesará la información.

Si se deja en blanco o se usa "#", los datos se enviarán a la misma página.

Atributo method

El atributo method define el método HTTP que se usará para enviar los datos. Los dos valores más comunes son:

  • GET: Los datos del formulario se añaden a la URL especificada en el atributo action. Es útil para formularios de búsqueda o cuando se quiere que la URL resultante se pueda marcar como favorita. Tiene un límite de tamaño y no es seguro para datos sensibles.
  • POST: Los datos del formulario se envían en el cuerpo de la solicitud HTTP. Es más seguro para enviar información sensible (como contraseñas) y no tiene límite de tamaño. Es el método preferido para formularios que modifican datos en el servidor (como registros o publicaciones).

Ejemplo de Código

Aquí tienes la estructura más básica de un formulario:


<form action="/procesar-datos" method="POST">

  <!-- Aquí dentro irán todos los campos del formulario -->
  <!-- (Inputs, labels, textareas, botones, etc.) -->

</form>
